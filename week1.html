Week 1

Set up Bracket, Github, and publish your first github hosted website

  1. Download Brackets from Brackets.io if you don't have it installed yet.
  2. Go to Finder=>Document folder and create a new folder called WEBPROJECTS
  3. Open Brackets, create a new file called index.html, type "my first github website" and save it in the WEBPROJECTS folder created from the above step.
  4. Go to github.com. Create a new account, and choose to set up a free plan.
  5. After clicing "Finishing sign up", go to your email account and verify your email address.
  6. Open Github on your computer, sign into the account you just created on github.com
  7. Click the "+" sign on the upper left corner, choose "add", and the path Document/WEBPROJECTS

    Warning 1! The default repo setting is "create", make sure you click "add", which is the tab to the left of "create"

    Warning 2! When you add the path, it's very important that you click on the WEBPROJECTS folder icon just once, so that the folder icon is highlighted, but you can't see the files that's actually inside the WEBPROJECTS folder.

    8. Now, if you go to your WEBPROJECTS folder, you should see a subfolder named .git. This folder contains files that will help you perform version control automatically from this point on.
  8. In Github, type "first commit" in symmary, click commit button, and publish button at the top right corner
  9. Now, click branchtab at the top middle bar, and create a new branch called “gh-pages”.
  10. Make sure "gh-pages" is now selected as current branch.
  11. After "gh-pages" is created successfully, you can view your page on http://[github user name].github.io/[repo name].
    • for example, if you go to http://zhenzhenBMCC.github.io/WEBPROJECTS, you should see "my first github webiste" on that page.
    Congratulations! You just hosted your first website through Github Pages.
  12. Now, go back to Brackets at type something more, go to Github Changestab on the top, type "second commit", click Commit, thensync, your website should be updated automatically.
  13. Take a look at the Historytab of your github, you can see notes and history from all your past commits.

    If anything goes wrong at any point during development, you can always go to "history", choose a previous version and rollback to this old version. See picture below.
